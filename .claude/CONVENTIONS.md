# OneKit Development Conventions

This document defines the mandatory development conventions
for the OneKit project.

---

## Commit Messages

All commit messages must be written in English.

### Rules

- Use imperative mood
  - Good: "Fix crash"
  - Bad: "Fixed crash"

- Be concise and descriptive
- Do not end the message with punctuation

### Examples

Good:
- Fix race condition in OneKitCore cache
- Add iOS 17 availability checks

Bad:
- Update code
- Fix some issues
